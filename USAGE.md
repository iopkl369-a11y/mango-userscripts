# 사용법 · 셀렉터 참조 (개발/운영)

더망고 주문정보의 배송지를 마켓 배송지 폼에 **단축키 한 번**으로 옮긴다.
**마켓별로 브라우저를 분리**하므로, 각 브라우저에 해당 스크립트만 설치한다.

| 파일 | 설치 브라우저 | 기능 |
|---|---|---|
| `mango_order_musinsa.user.js` | **Edge 전용** | 무신사 배송지 Alt+A 자동입력 (+카카오 우편번호) |
| `mango_order_ssg.user.js` | **Brave 전용** | SSG 배송지 Alt+A 자동입력 (+우편번호·배송요청사항) / Alt+D 진단 |
| `mango_order_naver.user.js` | **Chrome/Brave** | 네이버페이 주문서 Alt+A 자동입력(best-effort) / Alt+D 진단 |

> 더망고 주소 '담기'는 두 스크립트 모두에 들어 있다 — 마켓을 여는 그 브라우저에서
> 더망고도 함께 열어야 배송지가 담긴다(브라우저별로 저장소가 분리되므로).

## 사용 — 무신사 (Edge)

1. **더망고**에서 발주할 주문의 **주문정보**를 연다.
   → 우상단에 `✓ 주소 담김` 배지가 뜨면 배송지가 저장된 것.
2. **무신사** 상품에서 옵션·구매하기는 평소대로 직접 진행.
3. 주문서에서 **배송지 변경 → 배송지 추가하기**로 입력 폼을 연다.
4. **Option+A**(Mac) / **Alt+A**(Win) 한 번 →
   - 수령인·연락처(안심번호 050 → 회사번호 자동 대체)·상세주소(괄호 정리) 입력
   - 배송요청사항 직접입력
   - '주소 찾기' 자동 실행 → 카카오 검색·도로명 결과 자동 선택
5. 우편번호/도로명이 채워졌는지 눈으로 확인 후 **저장하기 → (목록에서 선택) → 변경하기**.
6. 금액·옵션·주소 최종 확인 후 **직접 결제**. (스크립트는 결제를 절대 누르지 않음)

## 사용 — SSG (Brave)

0. **더망고**도 Brave에서 함께 열어 주소를 담는다(배지 확인).
1. SSG 주문서에서 **배송지 입력/추가 폼**(`shpplocForm` 팝업)을 연다.
2. **Alt+A** 한 번 →
   - 수령인(`#rcptpeNm`)·주소별칭(`#address_alias`, 수령인 이름)·상세주소(`#address_detail`, 괄호 정리) 입력
   - 휴대폰은 **SSG는 항상 회사번호로 고정**(`#hpno1`+`#hpno2`, 값은 ⚙망고설정 `companyPhone`)
   - 우편번호 검색 팝업이 열리면 **주소의 괄호 `(금곡동, …아파트)`를 떼고** 검색어 자동 입력·'찾기' 실행 → **도로명 결과 자동 선택** → 팝업 **상세주소 입력 + '저장'**
   - 폼으로 돌아오면 폼 **'저장'** 자동 클릭 → 배송지 관리 목록에서 **해당 주소(이름+우편번호) 자동 선택 + '배송지 변경'**
   - **배송지 변경 후** 결제화면으로 돌아오면 **'택배배송 요청사항'을 '직접 입력'으로 선택하고 배송메모를 자동입력**(50자까지, 메모 있을 때만)
3. 금액·주소 최종 확인 후 **직접 결제**.

> 목록에서 자동선택이 실패하면(이름+우편번호 매칭 안 됨) 멈추고 알림 → 직접 선택·변경.

> 우편번호 팝업이 자동으로 안 열리면 폼의 **'우편번호 검색'** 칸을 직접 클릭(검색어는 이미 준비됨).

## 사용 — 네이버페이 (Chrome/Brave)

네이버페이 배송지(`orders.pay.naver.com/order/delivery`)는 SPA(같은 URL에서 화면 전환).

0. **더망고**도 같은 브라우저에서 열어 주소를 담는다(우하단 `✓ 주소 담김` 배지 확인).
1. **네이버페이 주문서**의 배송지 화면(배송지 목록 또는 입력 폼)에서 **Alt+A** 한 번 →
   - 목록이면 **'배송지 신규입력'** 자동 클릭 → 네이버는 **주소검색 화면이 먼저** 뜬다
   - **주소검색**: 검색어(괄호 `(송도동, …)` 제거) 입력·검색 → 결과행의 **'선택' 버튼** 클릭
   - **'상세주소를 알려주세요'(ⓑ)**: 상세주소 칸을 **클릭한 뒤** 입력(`#address-detail`, 괄호 정리)
     → **'확인'** 클릭. ※ 상세주소는 **이 단계에서만** 입력한다.
   - **'…확인 후 저장해주세요'(ⓒ)**: 수령인(`#receiver`)·연락처(`#contact-1`, 안심번호 050 **그대로**)·
     배송지 별명(`#delivery-name`=수령인)만 입력(**상세주소는 건드리지 않음**). 네이버는 **한 칸 입력 후
     '빈 곳'을 클릭해야 다음 칸이 떠서**, 필드마다 클릭→입력→빈곳클릭으로 진행한다.
   - **'저장하기'**가 활성화되면 자동 클릭 → 저장 후 목록에서 **해당 주소(이름+우편번호)의 '선택'** 클릭
2. (선택) 주문 페이지로 돌아오면 **배송메모**: 더망고 배송요청사항이 있을 때 **Alt+A 한 번 더** →
   '배송메모를 선택해주세요' → 모달 **'직접 입력하기'** → 텍스트박스에 배송요청사항 자동 입력.
   (메모가 없으면 건너뜀)
3. 우편번호/도로명·상세주소가 맞는지 눈으로 확인 후 **직접 결제**. (스크립트는 결제를 절대 누르지 않음)

> 입력 핵심: 네이버 React 폼은 저장 검증값을 **onBlur에 커밋**한다 → 스크립트는 값 주입 후
> `onChange`·`onBlur` 핸들러를 **직접 호출**해 인식시킨다(`reactFireProp`/`typeReact`).

> **저장/적용 버튼이 자동으로 안 눌리면** 그 화면에서 **Alt+D**(전체 버튼 덤프)를 떠서 전달 →
> 정확한 라벨로 고정한다. 주소 자동선택 실패(검색 0건/특이주소) 시 멈추고 알림 → **직접 선택**.

### 진단(Alt+D)
폼 구조가 바뀌어 자동입력이 안 먹을 때: 해당 화면에서 **Alt+D** → 검은 패널에 input 이름·버튼·iframe 덤프가 뜨고 자동 전체선택됨 → ⌘C로 복사해 전달.

## 동작 안 할 때

- "담긴 주문이 없어요": 더망고 주문정보를 먼저 열어 배지를 확인.
- "배송지 추가 폼이 안 보여요": '배송지 변경 → 배송지 추가하기'로 폼을 연 뒤 다시 Alt+A.
- 카카오 결과가 자동 선택 안 되면(주소 특이/검색 0건): 사람이 직접 선택 → **수동확인** 건.

## 간단입력 — 주문번호·금액·간단메모 (`mango_memo.user.js`)

발주 후 더망고 발주정보 폼(`상품주문번호·구입금액·간단메모`)을 **단축키 한 번**으로 채운다.
주소입력 스크립트와 **병행 설치**(같은 브라우저에 둘 다 설치 가능 — 단축키가 겹치지 않음).

- **출처(무신사/SSG/네이버)**: 평소처럼 **주문번호·금액·결제일시를 복사**만 하면 자동 캡처된다
  (형식으로 분류: 긴숫자·`날짜-영문` 식별자=주문번호, 콤마/짧은숫자=금액, 날짜=결제일시). 우하단 `캡처:…` 토스트.
  - **Cmd+C·우클릭복사**는 `copy` 이벤트로 잡고, **네이버 '주문번호 복사하기' 버튼**(클릭 시
    `clipboard.writeText`)은 **그 복사 버튼을 눌렀을 때만** 클립보드를 읽어 캡처한다(일반 클릭은 무시).
    버튼 판별: 클래스 `OrderNumber_button-number…` 또는 `복사` 문구+8자리↑ 숫자. **최초 1회 읽기 권한 허용** 필요할 수 있음.
- **더망고 주문목록(`admin_getorder.php`)**: 칸(상품주문번호·구입금액·간단메모)이 **주문 행마다** 있는 화면.
  우하단 **패널**에서 결제자 코드(자음/모음) 버튼을 한 번 고른다(한 명의 작업 동안 고정).
  **코드를 고르면 패널이 최소화**(코드·이름·캡처상태·채우기만 표시)된다 → 코드 칩(`ㅏ … ▸`)을
  누르면 다시 펼쳐 변경. 패널에 `✓ 주문번호/금액/일시` 캡처 상태가 보인다.
- **채울 주문 행의 칸(예: 상품주문번호)을 한 번 클릭** → **Alt+Q**(또는 패널 '채우기'): 그 행의
  상품주문번호·구입금액 입력 + 간단메모에 **기존 메모는 그대로 두고 맨 아래에 `코드 + 결제일시` 추가**.
  **저장/제출은 사람이 직접**(스크립트는 안 누름).
- **주문상태(드롭다운)**: 행 맨 앞 `[쿠팡 | …]`이면 `배송지입고완료`, 그 외(스마트스토어 등)는 `해외현지배송중`.
  **쿠팡 건만** 추가로 택배사 드롭다운 `삼성SDS` 선택 + 국내송장번호 **랜덤 12자리** 입력.
  ※ 더망고가 드롭다운 onchange에 자동 저장하면 그 시점에 반영될 수 있다(결제 아님).
- 캡처가 안 됐으면(일부 사이트는 JS 복사) 채울 때 **현재 클립보드 1값**으로 폴백 보충.
- 칸은 placeholder로 찾는다(`상품주문번호`/`구입금액`/행의 `textarea`=간단메모). 못 찾으면
  **Alt+D**(또는 '진단')로 그 행의 input/textarea `placeholder·name·id`를 덤프해 확인.
- 코드표(결제자 9건)는 코드에 두지 않고 **⚙망고설정(`mango_config`)** 에서 읽는다(`localStorage['mango_config'].codes`).

### 출처 사이트 단축키 — 한방캡처(Opt+W)·진단(Opt+S)

개별 Cmd+C 복사 대신 **단축키 한 번**으로 주문번호·금액·결제일시 3값을 동시 캡처(Mac=Option, Win=Alt).

- **무신사**: 주문상세에서 **Opt+W** → URL의 주문번호로 **거래명세서(`payment_receipt`)를 백그라운드 fetch**
  (동일 출처라 팝업·권한 불필요) → `dl.financial-form__list`의 `dt`(라벨)·`dd`(값)에서 결제일시(첫 날짜형식)·
  결제금액('결제'+'금액' 라벨)을 읽어 3슬롯 캡처. 거래명세서 팝업에서 직접 Opt+W도 가능(그 페이지 파싱).
- **네이버**: 주문완료 페이지에서 **Opt+W** → URL의 주문번호 + (접혔으면 주문상품 행 자동 펼침) **'결제상세' 제목을
  앵커로 '승인일시'까지 포함하는 최소 컨테이너**를 잡아 그 안의 금액(첫 `…,…원`=최종결제액)·승인일시를 캡처.
  React 해시 클래스 대신 한글 텍스트 앵커라 빌드 변화에 견딤.
- **SSG**: 주문상세(`orderInfoDetail.ssg`)에서 **Opt+W** → URL `orordNo`로 주문번호(하이픈 표시형 `20260607-4FB650`),
  **'결제내역' 칸의 실결제수단 금액**(없으면 '총 결제금액' 폴백), 결제일시는 **카드영수증 팝업
  (`orderInfoCardReceiptPopup`)을 백그라운드 fetch**해 시간 포함(`2026.06.07 15:50`, `-`→`.` 통일)으로 캡처.
  카드 외 결제수단이면 영수증 URL이 달라 결제일시가 상세의 날짜(시간없음)로 폴백될 수 있음.
- **Opt+S 진단**: 현재 화면의 '값 후보(형식분류)'·'라벨 후보'를 CSS경로와 함께 검은 패널에 덤프(자동 전체선택→⌘C).
- 기존 Cmd+C 개별 복사 캡처는 폴백으로 유지. Opt+W/Opt+S는 주소 스크립트의 Alt+A/Alt+D와 충돌하지 않음.

## 규칙·셀렉터 출처

- 상세주소 정리·안심번호 대체 규칙은 **별도 repo `mango_order`의 `src/musinsa_order.py`와 동일**(파이썬으로 6케이스 검증).
  한쪽을 고치면 양쪽을 같이 고쳐야 한다.
  단 **네이버는 안심번호(050x)도 그대로 입력**(회사번호 대체 없음) — 무신사/SSG와 다른 점.
- 더망고 필드: `buyer_name3·buyer_tel4·zipcode·addr1·addr2` (`mango_order`의 `config.yaml` `cafe24_selectors`).
- 무신사: `input[name=name|mobile|address2]`, 카카오 `#region_name`/`button.btn_search`/`.link_post`.
- 네이버페이: 폼 `#receiver`/`#contact-1`/`#delivery-name`/`#address-detail`,
  주소검색 결과 `li.AddressSearchList_item__wuxp8`>`p.AddressSearchList_address__4OJzp`,
  폼 저장 버튼 `저장하기`(`ButtonBox-module_article__b5G1-`), 중간단계 `확인`,
  목록 `li.DeliveryList_item__RmUJU`(이름 `.DeliveryList_name__EDWSs`), 항목 적용 버튼 `선택`.
- 네이버 입력 특이점: ①값 통째 주입을 무시 → **한 글자씩 InputEvent 타이핑**(`typeReact`),
  ②`_valueTracker` 강제 mismatch 필요, ③**마지막 글자는 추가 키 입력 후에야 인정** → 끝에 더미 공백 입력→삭제,
  ④한 칸 입력 후 **'빈 곳' 클릭**해야 다음 칸이 뜸(`clickEmptyArea`).
